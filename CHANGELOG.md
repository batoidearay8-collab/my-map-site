# 変更履歴 (Changelog)

本プロジェクトのバージョンごとの変更点を記録します。
形式は [Keep a Changelog](https://keepachangelog.com/ja/1.0.0/) に概ね準拠し、[セマンティック バージョニング](https://semver.org/lang/ja/) を採用しています。

## [10.23.0] - 2026-07

### 追加 (Added)
- **CSVファイル取り込みとマージモード**（Step 2 データ編集画面）。分担作業（Googleスプレッドシートで複数人が別々に入力→CSVダウンロード）を想定し、CSVファイルを直接取り込めるボタンを追加。取り込み方法を2つから選択できる。
  - **マージ（既定）**: 既存データに追記し、IDが同じ行は丸ごと上書き（地点はid、カテゴリはcategoryがキー）。既存行の位置は保たれ、新規行は末尾に追加。同一ファイル内にIDが重複する場合は後の行が勝つ。ID空欄の行はスキップして件数を通知。複数人のCSVを順番に流し込む運用が可能に。
  - **置き換え**: 現在のデータを捨ててファイル内容に置き換える（従来のテキストエリア反映と同じ意味）。
  - 取り込み結果は「追加◯件・上書き◯件（合計◯件）」のトーストで通知。取り込みはビルダーの「元に戻す」で1回で取り消せる。
  - 地点CSVとカテゴリCSVで別ボタン。取り込み後はテキストエリアも正規形に同期。
  - 研究モード計測に csv_import_pois / csv_import_cats カウンタを追加。

## [10.22.0] - 2026-07

### 追加 (Added)
- **ビルダーの作業時間計測モード（研究用・既定で完全無効）**。卒業研究のRQ1（作業時間）・RQ2（ユーザビリティ）のデータ収集用。
  - **入口**: ビルダーURLに `?research=1` を付けたときだけ同意ダイアログを表示（例: `#/builder?research=1`）。匿名の参加者ID（P01など）を入力し、同意した場合のみ計測開始。一般利用者には存在自体が見えない。
  - **計測内容**: ステップ別（取り込み/テンプレート/データ・画像/確認/公開/フロア作成）のアクティブ作業時間。マウス・キーボード入力が60秒以上途切れた時間はアイドル（休憩・離席）として除外。加えて操作カウンタ（元に戻す〔ビルダー/フロア〕・エラートースト・テンプレート適用・部屋テンプレ使用）。
  - **プライバシー設計**: データは localStorage のみに保存。**送信コードは一切なし**（`src/lib/builderMetrics.ts` に通信処理が存在しないことをソースで検証可能）。回収は画面左下の計測ウィジェットから CSV / JSON を手動ダウンロードして手渡し。
  - **参加者の権利**: いつでも「終了」で計測を停止でき、停止時に記録は全削除される。ブラウザを閉じても同意は保持され、次回起動時に新しいセッションとして計測を再開。
  - CSVは分析しやすい縦持ち形式（participant, session_id, start_iso, end_iso, kind, key, value）。Excel数式インジェクション対策済み。
  - 既存のビューワー用 researchLog（休眠中）とは独立したモジュールとして実装。ビューワー側の挙動に変更なし。

### 変更 (Changed)
- README に作業時間計測モードの説明（無効が既定・ローカル保存のみ・送信なし）を追記。

## [10.21.0] - 2026-07

### 追加 (Added)
- **フロア作成: 画像サイズを選べるように**。標準（2000×1400）に加えて、大（3000×2100）・特大（4000×2800）・縦長（1400×2000）の4プリセットから選択可能。サイズを変えても描いた部屋や記号はそのまま残る（小さいサイズに変更すると外にはみ出す場合がある旨のヒントを表示）。書き出しPNGと屋内マップへの取り込みサイズも選択に追従。
- **フロア作成: 部屋テンプレート**。教室・特別教室・階段・トイレ・昇降口・体育館・廊下の7種類を、ボタン1つでキャンバスに配置できる。色・サイズ・ラベルが最初から設定済みで、配置後は通常の部屋として移動・拡大縮小・回転・編集が可能。階段・トイレ・昇降口は対応する記号（🪜🚻🚪）も同時に配置され、元に戻す（Undo）1回で部屋と記号がまとめて取り消される。
- **フロア作成: 下絵の表示/非表示切り替え**。読み込んだ下絵（学校の平面図写真など）を一時的に隠して、描いた図だけを確認できるトグルボタンを追加。
  - ※下絵機能そのもの（写真を薄く敷いてなぞる・書き出しPNGには含めない）は以前から実装済みでしたが、変更履歴に未記載だったためここに明記します。下絵はセッション中のみ保持され、保存データには含まれません。

### 変更 (Changed)
- **フロア作成: 元に戻す（Undo）が記号にも対応**。記号の配置・移動・削除も履歴に記録され、Ctrl/Cmd+Z や「元に戻す」ボタンで取り消せるようになった（従来は部屋のみが対象）。

## [10.20.0] - 2026-06

### 修正 (Fixed)
- **屋内マップの初期表示でフロア全体が見えない不具合を修正**。固定の初期ズーム・最小ズームのせいで、画像が大きいとフロア全体が画面に収まらず、ズームアウトしても引ききれなかった。画像全体が画面に収まるズームを毎回自動計算し、最小ズームもそれに合わせて動的に下げるように変更。フロアを切り替えるたびにも全体表示に合わせ直すようにした。
- **フロア切替タブ（1F/2F/3F）で、右端を押すとハイライトが固定され他の階に移らない不具合に対処**。フロアのidが空または重複している場合でも、常にちょうど1つのタブだけが選択表示になるよう、選択判定を頑丈化した。

## [10.19.0] - 2026-06

### 修正 (Fixed)
- **テンプレート切り替えで屋内/屋外が変わらない不具合を本当に修正**。テンプレート定義が2か所（使われていない `StepTemplate.tsx` と、実際に使われている `BuilderPage.tsx`）に存在し、これまで前者だけを修正していたため反映されていなかった。実際に使われている `BuilderPage.tsx` 側の `applyTemplate` を修正し、テンプレートに応じて屋内/屋外モードが切り替わるようにした。
- **文化祭テンプレートのおすすめカテゴリを空に修正**。`BuilderPage.tsx` 側の文化祭テンプレートに残っていた `["屋内マップ", "複数フロア", "QRコード配布", "入退口表示"]` を空に変更。

### 変更 (Changed)
- 全テンプレートに屋内/屋外の別を定義（観光・ライブ・便利・祭り・アウトドア＝屋外、文化祭・防災＝屋内）。
- テンプレートのカードと選択時のプレビューに、屋内/屋外の表示を追加。

## [10.18.0] - 2026-06

### 変更 (Changed)
- **「CSVに反映」ボタンを画面上部の固定「保存」ボタンに一本化**。あちこちに散らばっていて分かりにくかった保存操作を、画面上部に常時表示される1つの保存ボタンに統合しました。

  - 上部の保存状態バッジをクリック可能な保存ボタンに変更。未保存の変更があるときは赤く強調＋点滅で「💾 保存する」、保存済みのときは「✅ 保存済み」を表示。これにより、保存し忘れて次に進んでしまう問題を防止。
  - ※ CSVテキストを直接手書き編集する上級者モードの取り込みボタンは、役割が異なるため残しています。

## [10.16.0] - 2026-06

### 追加 (Added)
- **フロア作成（FloorPlanEditor）第2段階**。
  - **記号の配置**: 階段・トイレ・出入口・EV・AED・案内の6種類の記号を、パレットから選んでキャンバスに配置できる。記号はドラッグで移動、ダブルクリックで削除。書き出しPNGにも反映。
  - **ラベル文字サイズの自動調整**: 部屋の大きさに合わせてラベルのフォントサイズを自動計算。小さい部屋でも文字がはみ出さず、大きい部屋では見やすく表示。プレビューと書き出しの両方に適用。
  - **元に戻す（アンドゥ）**: 部屋の追加・移動・拡大縮小・回転・色変更・削除・レイヤー変更・全クリアを1手ずつ取り消せる。「元に戻す」ボタンと Ctrl/Cmd+Z に対応（履歴は最大50手）。

## [10.15.0] - 2026-06

### 追加 (Added)
- **フロア作成（FloorPlanEditor）の部屋編集機能**。描いた後の部屋を自由に編集できるようになりました。
  - **移動**: 部屋をドラッグして平行移動（グリッド吸着対応）。
  - **拡大縮小**: 選択中の部屋の四隅ハンドルをドラッグ、またはサイドパネルの幅・高さ数値入力でリサイズ。
  - **角度の変更**: 部屋の上部の回転ハンドル、またはサイドパネルのスライダーで中心まわりに回転（吸着ONで15°刻み）。画面プレビューと書き出しPNGの両方に反映。
  - **レイヤーの変更**: 重なり順を「最前面・最背面・前へ・後ろへ」で変更。部屋が重なったときの上下関係を制御できる。

## [10.14.0] - 2026-06

### 追加 (Added)
- **簡易フロア作成機能（FloorPlanEditor）の強化**。第1段階として、以下を追加。
  - **色分けプリセット**: 選択中の部屋に色を設定できる（標準・教室=オレンジ・水回り=青・特別室=緑・施設=グレーの5種）。書き出すPNGと画面プレビューの両方に反映。文化祭の見取り図のように、部屋の種類を色で区別できる。
  - **グリッド吸着（スナップ）**: 部屋を描くときに50px単位のグリッドに自動で揃う。チェックボックスでON/OFF切替可能。手で並べてもガタつかず、見取り図らしく整列する。

## [10.13.0] - 2026-06

### 追加 (Added)
- **簡易フロア作成機能（FloorPlanEditor）**。外部の画像編集ソフトを使わなくても、AtlasKobo内でフロア画像（間取り図）を作れるようになりました。
  - キャンバス上をドラッグして長方形（部屋）を追加し、名前を付けられます。
  - 「この画像を使う」を押すと、Canvas APIでPNGに書き出し、既存のフロア画像の仕組み（`setBuilderAsset` + `imageWidthPx/HeightPx`）にそのまま取り込みます。ビューワーや経路探索のコードは変更不要。
  - 単一フロア・複数フロアの両方に対応（各フロアの「🖊️描いて作る」ボタンから起動）。
  - これまで「複数階を作るには外部でフロア画像を用意する必要があった」制約を解消。

## [10.12.0] - 2026-06
